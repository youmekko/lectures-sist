ERD - 테이블 구성

1. 큰 항목(엔티티) 구분
- 작은 항목(데이터)을 거느리는 부모 역할의 명사적 요소 추출.

예를 들어, 

 출판사(출판사명, 출판사전화번호)
 자바전문가(과정명)(자바(과목명), 오라클(과목명), ...)

 관리자, 강사, 학생 -> 대등한 관계
 
 강사 -> 과목에 대해서 참조
 학생 -> 과정에 대해서 참조

 강사 (이름, 주민번호, 전화번호, 강의 가능 과목) 

 입력, 출력 -> 데이터를 거느리는 명사적 요소?

2. 큰 항목을 테이블로 표현
-> 테이블은 사각형으로 그린다. 위에 제목처럼 테이블이름 표기.
-> 예를 들어, 강사 테이블

3. 작은 항목(데이터)를 컬럼으로 표현.
-> 예를 들어, 강사테이블에는 이름, 전화번호 컬럼 표기.
-> 테이블의 항목에서 중복된 자료가 있는 항목은 테이블 분리. 예를 들어, 지역 정보가 '서울'이 반복적으로 나온다면 지역 정보를 별도 테이블로 분리.
-> 테이블의 주인과 관련이 없는 항목은 테이블 분리. 예를 들어, 강사 테이블에 과목명 컬럼이 있다면?


4. 큰 항목들 간에 참조관계 파악해서 표기.
-> 예를 들어, 과정 테이블과 학생 테이블은 관계가 있다.
-> 테이블 두 개를 선으로 연결 표기.


5. 테이블마다 고유번호 저장을 위한 전용 컬럼 생성. 
-> 테이블명_id 컬럼
-> PK 컬럼은 테이블의 가장 상단에 위치시키고, 나머지와 구분하도록 경계선 또는 색상 등으로 표시.
-> PK와 연결되는 FK 컬럼은 색상 등으로 표시.
